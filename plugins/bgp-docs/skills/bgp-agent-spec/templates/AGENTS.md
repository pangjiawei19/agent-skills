## 分支策略

在开始任何代码修改之前，先确认当前所在分支：

- 如果当前分支是**功能分支**（如 `feature/*`、`fix/*`、`release/*`），则直接在当前分支上工作，**不要切换分支**。
- 如果当前分支是**主干分支**（`master`/`main`）或其他不适合直接开发的分支，则切换到 `<开发主分支，如 develop>` 后再开始工作。
