# 写规范的规范

本文件规定本项目规范（conventions）的写法。**新增或修改任何规范前必读**。

"规范"指 `AGENTS.md` 本体和 `docs/agents/` 下的分册。

读者是 AI Agent。一切规则的目的是让 AI 在尽量少的 context 开销下正确编码，不是给人读的教程。

---

## 零件货架（弱模板）

规范**不强制固定章节**，而是从下面的零件货架里按需挑。

### 核心 3 件（几乎每个规范都应有）

1. **位置 + 命名** — 目录、文件后缀、变量/类名格式、框架内 `name` 字段。AI 凭空编不出项目惯例，必须明写
2. **参考实例指针** — `` 参考 `src/tools/xxx-tool.ts` ``。AI 能读源文件，比内联示例更可靠、不会和代码漂移
3. **坑记** — AI 最容易犯、最容易被"优化掉"的反直觉规则。独立小节列出，不要散在行文中

### 按需 2 件

4. **新增 X 检查清单** — 流程步骤 ≥ 4 步、容易漏步时加。3 步以内规则本身顺着写就是清单
5. **结构/模式额外条款** — 参考实例 + 命名约定不足以说清"这个模块必须做的事"时才单独写（例："数据访问对象必须暴露 `connect()` / `disconnect()` 方法"）

### 不要放的 4 类

- **长示例代码**（> 5-6 行）→ 改成 `` 参考 `src/x/y.ts` ``
- **反面教材单独成段** → 合并到规则或坑记；行内 `// ❌ ... // ✅` 对比 OK，但限在小段内
- **纯背景/动机叙述** → 规则一行 + `**Why:** 一句原因` 紧跟其后
- **散落各处的命令片段** → 收到 `AGENTS.md` 顶部"常用命令"统一放

## 形式统一

下列几项与 AI 的定位/解析直接相关，必须统一：

| 零件 | 格式 |
|------|------|
| 小节标题固定词 | `### 命名约定` / `### 新增 X 检查清单` / `### 坑记`（不用"命名规则""步骤""注意事项"等同义词混写） |
| 命名约定呈现 | 表格 `\| 类型 \| 格式 \| 示例 \|`，不用散文式描述 |
| 参考实例句式 | `` 参考 `src/x/y.ts` ``（不用"见""样板见""参照"） |
| 检查清单结构 | 小节标题 `### 新增 X 检查清单` + 数字列表 |
| Why 行格式 | `**Why:** 原因一句话`，紧跟在规则后 |

**不强求**的：规则语气（"必须/应该/不要"按场景用）、小节缩进层级（`###` vs `####` 随复杂度定）、具体行文风格。

## 内联 vs 外链

### 判定标准

- **主题独立性**为主：能不能脱离 AGENTS.md 独立读？能 → 抽到 `docs/agents/`；不能 → 内联
- **超过 ~120 行**强制抽出（兜底，防 AGENTS.md 单文件过长）

### 外链引用方式

不使用 `@` 导入，改用**索引 + 触发条件**。**Why:** 抽离的本意是减 AGENTS.md 的 context 开销，`@` 会把内容自动注回，等于 token 挪了个地方没真减。

索引条目格式：

```
- X 规范 → docs/agents/x.md（<触发条件>）
```

**触发条件**要用具体动作，让 AI 做模式匹配。例：
- `修改 <对应目录> 下任意文件前必读`
- `写或改测试前必读`
- `新增或修改任何规范前必读`

不要写"了解 X 时参考""想深入可读"这种没动作的指引。

## 新增规范检查清单

新增一条规范时：

1. 判定内联 or 抽出（用"内联 vs 外链"章的标准）
2. 内联：写到 AGENTS.md 对应位置 + 在"规范索引"加锚点链接
3. 抽出：新建 `docs/agents/<name>.md` + 在"规范索引"加外链条目（含触发条件），**不用 `@` 导入**
4. 自查零件形式（用"形式统一"章的表格对照）
5. 自查有没有踩"不要放的 4 类"

## 修改现有规范检查清单

修改既有规范时：

1. 改内容
2. 检查阈值穿越：内联修改后超 ~120 行 → 抽出；外链精简后主题不再独立且行数 < 120 → 并回 AGENTS.md
3. 索引如有变动同步更新（锚点 ↔ 外链互换）
4. 不要顺手整改不相关的历史不一致之处（另开工作单做）

## 位置变更操作

- **抽离**（内联 → 外链）：AGENTS.md 删除对应章节 → 建 `docs/agents/<name>.md` → 索引条目由锚点改外链（加触发条件）
- **合并**（外链 → 内联）：删 `docs/agents/<name>.md` → 回填 AGENTS.md 对应位置 → 索引条目由外链改锚点

## 模块规范文件最小骨架

新增模块规范时，直接复制本骨架到 `docs/agents/<module>.md` 后填空：

````markdown
# <模块> 规范（`<src/路径>`）

<一句话说明本模块解决什么问题、什么时候必须创建对应文件。>
````
