# infoq-digest - InfoQ 资讯抓取技能

🦐 抓取 InfoQ AI&大模型频道最新资讯，输出富字段格式化列表

---

## 触发条件

用户说出以下或语义相近的表述时触发：
- "抓取 InfoQ 最新 AI 资讯" / "InfoQ AI 大模型"
- "拉一下 InfoQ 最近 N 条" / "InfoQ AI&LLM 频道"
- "抓 InfoQ 资讯并推送飞书"
- Cron 定时任务（见下方配置示例）

---

## 功能

1. 抓取 InfoQ AI&大模型频道（https://www.infoq.cn/topic/AI&LLM）最新资讯
2. 每条含完整元数据：标题、描述、作者、译者、时间、热度、链接
3. 按时间倒序取前 N 条（默认 15，最多 50）
4. 可选推送到飞书

---

## 使用方式

### 基础用法
```
抓取 InfoQ 最新 AI 资讯
```

### 自定义条数
```
抓取 10 条 InfoQ AI 资讯
```

### 推送飞书
```
抓取 InfoQ 资讯并推送到飞书
```

### Cron 定时任务
```yaml
sessionTarget: "main"
payload:
  kind: "systemEvent"
  text: "执行 infoq-digest 技能，抓取 InfoQ AI&大模型频道最新 15 条资讯并推送到飞书"
```

## 实施步骤

### 1. 抓取页面 HTML（首选）

使用 `web_fetch` 拉取目标 URL：

```
web_fetch "https://www.infoq.cn/topic/AI&LLM"
```

> ⚠️ URL 中的 `&` 在 shell 场景需转义或整串用引号包裹，避免被解释为后台执行符。

InfoQ 是 SSR 页面，列表通常已内联在初始 HTML 中，可直接从响应中解析出所有 card。

### 2. 备选：浏览器快照

若 `web_fetch` 返回的 HTML 列表为空（可能 InfoQ 改为 CSR 渲染，或被风控拦截返回空壳页面），使用 `browser` 工具回退：

1. `browser` 打开 `https://www.infoq.cn/topic/AI&LLM`
2. 等待首屏加载（必要时滚动一次触发懒加载）
3. `browser.snapshot` 取 aria 结构化快照

### 3. 解析列表

无论哪种方式，InfoQ 列表页每条文章是一个 card，自上而下即为时间倒序。每个 card 一般包含：
- heading（level=6）：文章标题 + 绑定的文章链接
- 摘要段落：描述
- meta 区：作者 / 译者（可选）/ 相对时间 / 热度数值

### 4. 逐条提取字段

| 字段 | 必填 | 提取位置 |
|------|------|----------|
| 标题 | ✅ | heading 文本 |
| 链接 | ✅ | 标题绑定的 URL |
| 描述 | ✅ | 标题下方摘要段落 |
| 作者 | ✅ | meta 区人名 |
| 译者 | ⚠️ | meta 区若存在则附加，无则省略 |
| 时间 | ✅ | 相对时间字符串（"53 分钟前" / "13 小时前"） |
| 热度 | ✅ | meta 区末尾数字 |

**过滤规则（优先级从上到下）：**

1. **结构化过滤（主）**：一条 card 必须同时具备「描述 + 作者 + 时间 + 热度」四项才算文章，缺任一核心字段直接跳过。这是最稳的判断方式——导航、赞助位、专题入口、轮播横幅天然缺 meta，会自动被过滤掉，不依赖维护关键词表。

2. **URL 辅助白名单**：仅收录路径含 `/article/` 或 `/news/` 的 URL。若页面出现 `/minibook/`、`/presentation/`、`/video/` 等新内容类型，按需扩充；但 `/topic/`、`/u/`（用户主页）、`/conference/`（会议专区）、`/qconxxx` 永远不是单篇文章，直接排除。

3. **标题兜底黑名单（可选，会轮换）**：仅作为双保险，观察到的噪声项包括「首页 / AI 会议 / AI 课程 / AI 应用 / 报告」、「HarmonyOS」、「Snowflake」、以及含「企业动态 / 行业深度」字样的运营位。InfoQ 赞助位会定期轮换，不要把这份名单当真理——结构化过滤和黑名单冲突时，信任结构化过滤。

> 关键区分：规则 1 丢弃的是"本来就不是文章"的噪声；而"看起来是文章却缺字段"属于异常，走 [失败处理](#失败处理) 流程报错，不要静默跳过。

提取到 `count` 条后停止。

### 5. 格式化输出

```
### 1. 标题
**描述：** 摘要内容...
**作者：** xxx | **译者：** xxx | **时间：** 53 分钟前 | **热度：** 465
[🔗 原文链接](https://...)

---
```

- 译者缺省时去掉"| **译者：** xxx"整段，不要留空
- 开头加一行汇总：`**InfoQ AI&大模型 - 最新 N 条资讯** 🦐`

### 6. 推送飞书（可选）
若 `push=true`，使用平台提供的飞书消息工具推送完整输出。失败需打印错误，不要静默。

---

## 失败处理

- **字段缺失（已通过过滤规则 1）**：打印缺失字段名 + 文章 URL，不要用 "N/A" 或 "未知" 填充必填字段——card 明明长得像文章却缺字段，意味着页面结构变化，需要人工确认
- **结构化过滤误伤**：若通过规则 1 后剩余条数 < 请求 `count` 的 50%，提示"过滤可能过严，建议检查页面结构是否改版"
- **抓取条数不足**：输出实际条数并提示原因（可能未加载完 / 需要滚动 / 被过滤规则误伤）
- **网络/页面错误**：重试 1 次；仍失败返回明确错误，不要静默降级
