# weekly-review - 周工作总结 Skill

**版本：** 1.0  
**创建时间：** 2026-04-07  
**作者：** 小虾米 🦐

---

## 📋 功能描述

基于每日任务记录，自动生成周工作总结飞书文档，包含工作任务、学习与成长、生活内容、关键词提炼、复盘反思等模块。

---

## 🎯 触发条件

当以下情况时使用此 Skill：

1. 用户说"做周总结"、"本周总结"、"周工作总结"
2. 每周日 18:00（后续可配置 Cron 自动触发）
3. 用户要求整理本周工作/学习/生活内容

## 🔄 执行流程

### 1. 收集每日记录

读取 `memory/YYYY-MM-DD.md` 文件，获取本周（周一→周日）的所有记录：

```bash
# 示例：读取本周每日记录
read memory/2026-04-01.md
read memory/2026-04-02.md
...
read memory/2026-04-07.md
```

### 2. 分类整理

将任务按三大类分类：

| 类别 | 识别规则 |
|------|---------|
| **💻 工作任务** | 项目开发、代码编写、会议沟通、文档交付、自动化任务等 |
| **📚 学习与成长** | 读书、技术调研、英语学习、课程学习等 |
| **🏠 生活内容** | 仅记录用户主动分享的内容（运动、做饭、家庭等） |

**边界说明：**
- 读《白夜行》→ 学习（提升文学素养）
- 纯娱乐电影 → 生活（如用户主动分享）
- 技术文档阅读 → 学习
- 运动健身 → 生活（如用户主动分享）

### 3. 提炼关键词

基于本周内容，提炼 3-5 个关键词，每个带简短说明：

**示例：**
```
1. **MCP 架构迁移** - 完成 EKS MCP 替换 Tools，生产验证通过
2. **Family-Assistant 收尾** - 文档、代码规范整理完毕，项目告一段落
3. **自动化稳定运行** - GitHub Trending、InfoQ 资讯每日推送无故障
```

### 4. 选取复盘事项

选取 1-3 件重要事情进行复盘，使用固定模板：

```markdown
### 事项 1：xxx

- **事实：** 计划了什么，实际完成了什么
- **原因/分析：** 为什么会有差异，原因是什么
- **改进措施：** 下次怎么做
```

**选取标准：**
- ✅ 有重大进展/突破的事
- ✅ 有计划与实际的差异
- ✅ 有重要经验教训
- ❌ 常规执行、无特殊事项的不复盘

### 5. AI 视角观察（可选）

仅在有新认知时填写：

- 观察到的协作模式/工作习惯
- 没有新认知时写"暂无新观察"

**边界：** 只观察协作模式、工作习惯，不评判个人选择

### 6. 统计工作量

按类别统计任务数：

| 类别 | 完成任务数 | 占比 |
|------|-----------|------|
| 💻 工作任务 | X+ | ~X% |
| 📚 学习与成长 | X | ~X% |
| 🏠 生活内容 | X | ~X% |
| **合计** | **X+ 项** | **100%** |

### 7. 创建飞书文档

使用 `feishu_create_doc` 创建文档：

```yaml
title: "周工作总结 (2026-XX-XX ~ 2026-XX-XX)"
markdown: <完整内容>
folder_token: <"个人规划"目录的 token>
```

**注意：** "个人规划"目录 folder_token = `BmD4fnF5dlOr64ddqn7cVheQnub`

### 8. 发送飞书通知

使用 `feishu_im_user_message` 发送通知：

```json
{
  "msg_type": "text",
  "content": "{\"text\": \"老大，本周总结已生成：[文档链接]\"}"
}
```

## ⚠️ 注意事项

1. **周的定义：** 周一 → 周日（7 天），跨周任务要准确计算
2. **生活内容边界：** 仅记录用户主动分享的，不主动追问
3. **复盘粒度：** 选取 1-3 件重要事情，不事事复盘
4. **文档位置：** 固定存放在"个人规划"目录
5. **执行时间：** 周日 18:00（北京时间）
6. **实时记录：** 依赖每日 `memory/YYYY-MM-DD.md` 记录，需确保平时记录完整

## 📝 使用示例

### 手动触发

```
用户：做一下本周总结
助手：好的，正在整理本周（X 月 X 日-X 月 X 日）的记录...
     （读取每日 memory 文件）
     （分类整理、提炼关键词、选取复盘事项）
     （创建飞书文档）
     （发送通知消息）
助手：老大，本周总结已生成：[文档链接]
```

### 自动触发（后续配置 Cron）

```yaml
schedule:
  kind: "cron"
  expr: "0 18 * * 0"  # 每周日 18:00
payload:
  kind: "systemEvent"
  text: "执行 weekly-review skill，生成本周工作总结"
```
