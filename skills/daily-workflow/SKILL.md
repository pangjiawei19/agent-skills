# 日常工作流程

## 核心流程

```
🌅 早上老大说"开工"
    ↓
我主动问老大：今天要做什么？有几件重点事？
（不从 git / 代码 / 昨日记录推断，一律问老大）
    ↓
☀️ 白天：老大主动说"完成了 xxx"
    ↓
我简单记录，不追问细节（降低干扰）
    ↓
🌙 老大说"收工"
    ↓
我拉清单 → 老大确认/补充 + 告诉我今天有没有其他重要事
    ↓
我推荐 1-3 件值得复盘的事 → 老大确认或指定其他
    ↓
按模板复盘：事实 → 原因 → 改进措施
    ↓
沉淀到 memory/YYYY-MM-DD.md
```

## ⚠️ 禁止编造原则（核心红线）

- **只记录老大亲口说的内容**，不推断、不扩写、不脑补
- 开工拉清单 → 问老大，禁止从 git log / 代码改动 / 昨日未完成项自动推断今日任务
- 白天记录 → 老大说什么记什么，不为了"看起来饱满"补充细节
- 收工复盘 → 某项没信息就写少，不要用"应该、可能、大概"填空
- 与 `weekly-review` 保持一致红线：内容不足直接写少，不编造

## 任务记录规则

| 环节 | 执行方式 |
|------|----------|
| **开工清单** | 我主动问老大今天做什么，不自行推断 |
| **任务记录** | 老大主动告诉我，我简单记（不追问） |
| **追问补细节** | 晚上"收工"后一起补 |
| **复盘触发** | 我按重要性推荐 1-3 件 + 老大可指定 |
| **实时写文件** | 每次说"完成了 xxx"，立即更新 memory 文件（防会话重启丢失） |

## 复盘模板

```markdown
### 事项 N：{事项名称}

**事实：**
- 计划了什么
- 实际完成了什么
- 关键时间节点

**原因/分析：**
- 为什么会有差异
- 原因是什么（客观/主观）
- 有什么意外发现

**改进措施：**
- 下次怎么做
- 需要沉淀到什么文件/流程
```
