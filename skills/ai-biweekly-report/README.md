## 🚀 快速开始

### 手动触发

```
请生成 2026 年 3 月上半月 AI 工程半月报（3 月 1 日 -15 日）
```

### 自动化触发

配置 Cron 后自动执行：
- **每月 16 号 9:00** → 生成上半月报（1-15 日）
- **次月 1 号 9:00** → 生成下半月报（16-月底）

## 📋 技能功能

### 核心能力

1. **多数据源搜索**：8 大数据源，50-80 条候选资讯
2. **日期严格核实**：逐条 web_fetch 确认时间范围
3. **深度分析**：What/How/Where/Why 四维度分析
4. **原文链接**：每条资讯附带可点击原文 URL
5. **工程视角**：面向后端工程师转型 AI Engineer

### 输出内容

- **飞书文档**：完整深度分析报告
- **消息摘要**：Top 10 速览 + 文档链接
- **行动建议**：技术栈迁移、架构思考、学习重点

## 📊 资讯构成

| 类别 | 占比 | 说明 |
|------|------|------|
| 学术界 | 30% | 有开源代码、新架构、RAG/微调优化 |
| 产业界 | 50% | 新模型 API、框架更新、向量数据库 |
| 高价值实践 | 20% | GitHub 热门、工程最佳实践 |

## 🔧 配置选项

### 时间范围

```yaml
report_period: "first_half"  # 上半月 (1-15 日)
report_period: "second_half" # 下半月 (16-月底)
report_period: "custom"      # 自定义范围
```

### 交付格式

```yaml
delivery_format: "feishu_doc"  # 飞书文档
delivery_format: "markdown"    # Markdown
delivery_format: "both"        # 两者都生成
```

## 📈 执行流程

```
Step 1: 多数据源搜索 (8 大数据源)
         ↓
Step 2: 逐条 web_fetch 核实日期
         ↓
Step 3: 按优先级筛选 Top 10
         ↓
Step 4: 深度分析 (What/How/Where/Why)
         ↓
Step 5: 生成转型指导建议
         ↓
Step 6: 创建飞书文档 + 发送摘要
```

## ⚠️ 注意事项

1. **时间范围严格核实**：每条资讯必须确认在指定时间范围内
2. **原文链接必附**：方便用户快速查看原文
3. **宁缺毋滥**：高质量资讯少于 10 条时不凑数
4. **避免重复**：半月报周期确保不重复报道
