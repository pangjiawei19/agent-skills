# AI 工程半月报技能

## 技能名称

`ai-biweekly-report` - AI 工程半月报生成器

## 技能描述

自动搜集、筛选、分析 AI 领域关键资讯，生成面向后端工程师转型 AI Engineer 的深度分析半月报。

## 适用场景

- 需要定期了解 AI 领域最新工程动态
- 从后端工程师视角过滤技术噪音
- 关注工程可行性、API 调用、系统架构
- 需要可追溯的原文链接和深度分析

## 触发条件

用户请求：
- "生成 AI 工程半月报"
- "搜集本周/本月 AI 资讯"
- "AI 行业周报/月报"
- "AI 工程动态"

## 执行流程

### Step 1: 多数据源搜索（扩大候选池）

搜索 8 大数据源，预期 50-80 条候选：

| 数据源 | 搜索关键词 | 预期产出 |
|--------|-----------|----------|
| Arxiv | `cs.CL cs.LG cs.AI March 2026` | 10-15 条 |
| Hugging Face | `new model release March 2026` | 5-8 条 |
| TechCrunch | `AI startup funding March 2026` | 5-10 条 |
| Hacker News | `AI LLM March 2026` | 10-15 条 |
| 厂商博客 | `OpenAI Anthropic Google AI March 2026` | 5-8 条 |
| GitHub Trending | `AI machine learning trending` | 5-10 条 |
| 向量数据库 | `Pinecone Weaviate Milvus March 2026` | 3-5 条 |
| AI 工程实践 | `RAG production deployment 2026` | 5-8 条 |

**工具**：`web_search`（每次 count=10，分批次搜索）

### Step 2: 逐条核实日期（web_fetch）

对每条候选资讯：
1. 用 `web_fetch` 抓取原文
2. 从原文中提取发布日期
3. 过滤掉不在时间范围内的内容
4. 记录原文链接

**时间范围**：
- 上半月报：当月 1-15 日
- 下半月报：当月 16-月底

**预期通过率**：20-30% → 最终保留 10-15 条

### Step 3: 按优先级筛选 Top 10

筛选标准：
1. **产业界优先**（50%）：新模型 API、框架更新、向量数据库、云厂商服务
2. **学术界精选**（30%）：有开源代码、新架构、提升 RAG/微调效果
3. **高价值实践**（20%）：GitHub 热门项目、工程化最佳实践

### Step 4: 深度分析（逐条处理）

每条资讯按以下格式输出：

```markdown
### [序号] 标题

**来源**：数据源 | 发布日期
**原文链接**：URL

- **核心情报 (What)**：一句话概括发生了什么

- **技术原理解析 (How)**：
  - 技术逻辑/实现机制
  - 关键创新点
  - 性能指标

- **应用场景 (Where)**：
  - 可以做什么产品
  - 解决什么痛点

- **工程化价值 (Why)**：
  - 对开发者意味着什么
  - 成本/延迟/架构影响
  - 权衡和注意事项
```

### Step 5: 转型指导与行动建议

包含三个部分：

#### 技术栈迁移建议
| 项目 | 推荐理由 | 优先级 | 链接 |
|------|----------|--------|------|
| 项目名 | 为什么值得 Clone | 高/中/低 | GitHub 链接 |

#### 架构思考
- 数据库选型影响
- 微服务拆分建议
- API 网关/缓存层设计

#### 学习重点
- 下周/半月建议补充的知识
- 推荐学习资源

### Step 6: 生成飞书文档

1. 用 `feishu_create_doc` 创建完整文档
2. 用 `message` 发送摘要消息（含文档链接）

## 配置参数

### 时间范围
- `report_period`: "first_half" | "second_half" | "custom"
- `start_date`: YYYY-MM-DD（custom 模式）
- `end_date`: YYYY-MM-DD（custom 模式）

### 内容比例
- `academic_ratio`: 0.3（学术界）
- `industry_ratio`: 0.5（产业界）
- `practice_ratio`: 0.2（高价值实践）

### 交付设置
- `delivery_format`: "feishu_doc" | "markdown" | "both"
- `send_summary`: true | false

## 错误处理

### 常见问题

1. **候选资讯不足 10 条**
   - 处理：有几条写几条，不凑数
   - 告知用户："本期高质量资讯较少，仅筛选出 X 条"

2. **web_fetch 失败**
   - 处理：尝试备用 URL 或跳过该条
   - 记录失败原因

3. **日期无法核实**
   - 处理：保守估计，归入"日期不明"类别
   - 在文档中标注

4. **飞书授权失败**
   - 处理：提示用户完成授权
   - 提供 Markdown 版本作为备选
