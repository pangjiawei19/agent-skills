# 技术 / 工具书 — 读后总结段模板

> 本模板是「📖 读后总结」段落的子结构，由 `+summary` 注入到 `reading/{key}.md` 里。
> 不要重复进度表、章节笔记、感想等主文件已有的内容，只放类型独有的回看提炼。

---

### 本书定位
- 一句话概括：解决什么问题、面向什么读者
- 前置知识要求
- 与同类书的区别

### 知识地图
- 核心概念之间的关系（列表或 Mermaid 图）
- 按主题分组的知识点清单

### 核心概念
- 表格：概念 / 定义 / 关键特征 / 易混淆点

### 实践要点
- 可落地的做法（表格：场景 / 做法 / 注意事项）
- 常见坑与反模式

### 个人洞察
- 与我当前工作 / 项目的结合点
- 颠覆了哪些旧认知
- 哪些地方和我的经验冲突

### 未解与待深入
- 尚未理解透的点
- 想进一步查阅的资料

### 总体评价
- 多维度评分（信息密度 / 实用性 / 可读性 / 时效性）

### 落地行动
- 接下来要做的 1-3 件具体事
