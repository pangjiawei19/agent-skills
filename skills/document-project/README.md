# Project Documentation Generator

自动从代码库生成全面项目文档的技能。

## 功能特性

✅ **项目概览分析** - 自动识别项目类型、技术栈和基础信息
✅ **领域模型提取** - 提取核心实体、关系和业务规则
✅ **业务流程梳理** - 追踪业务流程，生成序列图和状态机
✅ **架构结构分析** - 分析目录结构和代码组织模式
✅ **依赖关系整理** - 整理外部依赖和第三方服务集成
✅ **Mermaid 图表** - 自动生成 ER 图、序列图、状态图

## 生成的文档包含

1. **项目简介**
   - 项目类型和技术栈
   - 版本信息
   - 项目目标

2. **核心领域模型**
   - 实体关系图（Mermaid ER Diagram）
   - 每个核心实体的详细说明
   - 业务规则和约束

3. **业务流程**
   - 主要业务场景的序列图
   - 关键步骤说明
   - 异常处理逻辑

4. **项目结构**
   - 目录树和模块说明
   - 架构模式分析
   - 代码组织规范

5. **外部依赖**
   - 框架和库依赖清单
   - 第三方服务集成说明
   - 版本信息

6. **部署说明**
   - 环境要求
   - 配置步骤
   - 启动命令

## 最佳实践

### 何时使用

- ✅ 新接手项目，需要快速了解架构
- ✅ 项目缺少文档，需要补充
- ✅ 代码重构后，需要更新文档
- ✅ 团队新成员入职，需要项目介绍材料

### 生成后的优化

AI 生成的文档是一个很好的起点，但建议手动补充：

1. **业务背景** - 为什么要做这个项目？解决什么问题？
2. **架构决策** - 为什么选择这个技术栈？有哪些权衡？
3. **已知问题** - 当前存在的技术债务和待优化点
4. **性能指标** - QPS、响应时间等关键指标
5. **安全考虑** - 安全机制和注意事项

## 常见问题

### Q: 生成的文档不准确怎么办？

A:
1. 检查代码注释是否完善
2. 确保关键业务逻辑有清晰的命名
3. 手动补充 AI 无法推断的业务背景

### Q: 可以自定义文档模板吗？

A: 可以。修改 `SKILL.md` 中的输出格式部分，调整章节结构和内容重点。

### Q: 支持多语言文档吗？

A: 当前默认生成中文文档。如需英文文档，在调用时明确说明：
```
Generate project documentation in English
```

### Q: 大型项目（1000+ 文件）如何处理？

A: 技能会自动采用采样分析策略：
- 优先分析核心模块
- 每个模块选择代表性文件
- 使用并行分析提高效率

## 技术实现

该技能使用以下工具进行代码分析：

- **SemanticSearch** - 理解代码语义，查找特定模式
- **Grep** - 精确搜索注解、关键字
- **Glob** - 查找特定类型的文件
- **Read** - 读取关键文件内容
- **Task** - 并行分析多个模块
