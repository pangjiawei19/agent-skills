# 技术文章分析

抓取技术文章 URL，产出结构化分析报告，存储到飞书文档。

## 工作流程

### 1. 抓取文章

根据 URL 类型选择抓取方式：

- **普通网页**：`web_fetch` 获取正文
- **微信公众号**（`mp.weixin.qq.com`）：使用 `wechat-article-reader` skill
- **飞书文档**：`feishu_fetch_doc`
- **PDF**：`pdf` 工具

如抓取失败，尝试备用方式（浏览器截图、简化 URL 等）。

### 2. 分类定性（先于分析执行）

快速通读后，先判定文章类型，后续分析的深度和侧重由此决定：

| 类型 | 判定特征 | 分析侧重 |
|------|---------|---------|
| **新技术/框架** | 介绍新工具、新库、新平台 | 解决了什么问题、与现有方案对比、成熟度评估 |
| **教程/最佳实践** | 手把手教学、step-by-step | 实操性、适用场景、与当前技术栈兼容性 |
| **架构设计** | 系统设计、方案选型、演进过程 | 设计决策权衡、可扩展性、踩坑经验 |
| **观点/趋势** | 预测、评论、行业分析 | 论据充分性、数据支撑、与实际情况匹配度 |
| **论文解读** | 学术论文或论文导读 | 核心创新点、实验结果、落地可行性 |
| **快讯/资讯** | 短篇、信息密度低、新闻通报性质 | 仅做快速评估，不做深度分析 |

### 3. 深度阅读与分析

根据分类结果，动态选择分析模块。各模块标注 ✅必填 / ⚡按类型必填 / 💡选填：

#### a) 值不值得读 ✅

所有文章必填，这是最核心的决策依据。

- **评分（1-10）**：根据内容质量、创新性、对老大的实际价值综合打分
- **一句话理由**：为什么值得/不值得读，直击要害
- **实操成本**：落地该技术/方法需要的时间估算 + 前置条件

> **短路规则**：评分 ≤ 3 分的文章，只填 a) 和 b)，跳过后续所有模块，不值得花时间展开。

#### b) 核心摘要 ✅

所有文章必填。

- **一句话定义**：用最直白的后端技术语言（类比）说清这是什么
- **3 条关键结论**：最核心、最确定的观点，不要泛泛而谈
- **技术定位**：明确细分方向（如 RAG 基础设施、Agent 框架、Prompt 方法论、模型微调等）
- **标签**：2-5 个标签，如 `#RAG` `#Agent` `#向量数据库` `#Prompt工程`

#### c) 深度解读 ⚡

按文章类型侧重不同内容，快讯/资讯类跳过此模块：

- **技术要点拆解**：
  - 新技术/框架 → 核心机制 + 与现有方案差异对比
  - 教程/最佳实践 → 关键步骤 + 适用边界
  - 架构设计 → 设计权衡 + 演进动因
  - 观点/趋势 → 核心论据 + 数据支撑度
  - 论文解读 → 创新点 + 实验设计 + 结果可信度
- **与当前工作/学习的关联**：
  - 与老大正在进行的项目的关系（互补/可借鉴/有冲突/无关）
  - 对学习方向的帮助
  - 参考会话上下文中已加载的 MEMORY.md 和近期 `memory/YYYY-MM-DD.md`，获取最新项目状态和技术背景

#### d) 延伸 💡

有则写，没有不硬凑。快讯/资讯类和评分 ≤ 5 分的文章跳过。

- **风险与局限**：文章未提及的坑、边界条件、潜在问题
- **开放性问题**：值得深入思考的深层次问题（1-2 个）
- **建议延伸阅读**：相关对比文章或补充资料

### 4. 引用补全（评分 > 5 的文章执行）

文章中提到的论文、工具、框架、延伸阅读，自动搜索并附上可访问链接：

1. **提取引用项**：从文章内容和分析结果中收集所有提及的论文标题、项目名、技术名词
2. **逐项搜索**：使用 `web_search` 搜索每个引用项，优先找到：
   - 论文 → arXiv / Semantic Scholar / 原始发布页
   - 开源项目 → GitHub 仓库
   - 技术文档 → 官方文档链接
3. **验证链接**：用 `web_fetch` 确认链接可访问，标题匹配
4. **写入报告**：
   - 文中提及的论文/项目 → 在「深度解读」中以行内链接形式嵌入
   - 建议延伸阅读 → 在「延伸」模块中列出，附简短关系说明

> **原则**：搜不到的不硬编链接，标注「未找到公开链接」即可。宁可少一个链接也不要放一个 404。

### 5. 生成飞书文档

使用 `feishu_create_doc` 创建报告：

- **标题**：`[分析报告] 文章标题`
- **位置**：默认创建到「技术文章分析」文件夹（folder_token: `EUSCfWgKtlx8MWdpQGvc3meinKd`），URL: https://magictavern.feishu.cn/drive/folder/EUSCfWgKtlx8MWdpQGvc3meinKd
- **模板选择**：
  - 评分 ≤ 5 或快讯/资讯类 → `references/report-template-quick.md`（快速评估版）
  - 其余 → `references/report-template-full.md`（完整分析版）
- **Markdown 格式**：使用飞书 Markdown 语法

### 6. 返回结果

- 飞书文档链接
- 评分 + 一句话总结（方便快速决策）
- 询问是否需要进一步操作（如调整分析角度、批量处理其他文章等）

## 批量分析

当用户同时提供多个 URL 时：

1. 先逐个抓取
2. 每篇文章独立分析
3. 每篇文章生成独立的飞书文档
4. 最后汇总一份对比表（评分 + 标签 + 一句话定义），方便快速浏览

## 注意事项

- **宁缺毋滥**：文章质量差就直说，不要硬凑价值
- **评分要真实**：5 分以下说明质量不行，不要怕得罪作者
- **关联要有依据**：提到与老大工作的关联时，要具体到项目/方向，不泛泛而谈
- **延伸阅读要实用**：推荐的文章必须确实相关，不要随便列
- **时效性敏感**：技术文章如果超过 1 年，要指出可能的过时风险
