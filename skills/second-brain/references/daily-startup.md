# 开工 — daily 段第一动作

## 语义

用户说"开工"=今天正式开始。Agent 进入秘书模式（**非教练**），询问今日三类任务并立刻落盘到 `~/second-brain/daily/YYYY-MM-DD.md`。

## 启动流程

1. **加载 profile**（用 Read 工具，顺序如下；任一文件不存在时跳过，不报错）：
   - `~/second-brain/profile/user.md`
   - `~/second-brain/profile/soul.md`（决定说话风格、忌讳）
   - `~/second-brain/profile/patterns.md`（显式偏好和归纳规律）
   - `~/second-brain/profile/memory.md`
2. **不读** `~/second-brain/raw/` 和 `~/second-brain/daily/` 历史——开工只问今天，不被历史牵着走。
3. **检查今日 daily 文件**：路径 `~/second-brain/daily/YYYY-MM-DD.md`（基于今天日期）。
   - 存在 → 告诉用户"今天已经开过工了，要加新任务吗？"。用户答"加 X" → 走"继续开工"分支用 Edit 追加 `- [ ] (TODO) X`（见下方落盘格式要点）；用户答"没了" / "看看就好" → 结束
   - 不存在 → 创建模板文件（见下方），然后开始问任务

## 询问任务

问一句让用户一次性给出今天的工作 / 学习 / 生活任务，按下方"落盘格式"用 Write 工具一次写入文件。

**禁止从 git log、代码改动、昨日记录、日历事件等推断**——只记用户亲口说的（daily 红线 2 + 5）。

## 落盘格式

文件首次创建时，内容如下：

```markdown
# YYYY-MM-DD

## 📋 今日任务清单

### 💻 工作
- [ ] (TODO) {用户说的工作 1}
- [ ] (TODO) {用户说的工作 2}

### 📚 学习
- [ ] (TODO) {用户说的学习 1}

### 🏠 生活
- [ ] (TODO) {用户说的生活 1}
```

要点：

- 三大类（工作 / 学习 / 生活）一直保留，即使某类用户没讲任何任务——空类保留标题，下方留空（不写"无"，等收工再决定）
- 用户说出口即视为确认，**无需复述确认**（daily 红线相关）
- 首次创建用 Write 工具
- "继续开工"分支：用 Edit 工具在对应分区末尾**追加**新条目 `- [ ] (TODO) 描述`（不覆盖已有条目）；若 Edit 锚点不唯一（多个分区末尾结构相同），退化为 Read 整文件 + Write 整文件重写，并保留所有已有条目和它们的勾选状态

## 边界情况

- **用户三类任务都没讲清楚**：仍创建文件、三类标题保留为空，提醒用户"如果还没想好就先这样，回头随时说'再加一个 xxx'我加进去"
- **用户白天突然说"再加一个 xxx 到工作清单"**：找到对应分区末尾追加 `- [ ] (TODO) 描述`（计划内追加，不是计划外，因为未完成）

## 不做什么（红线提醒）

- 不脑补"用户今天可能也会做 X"
- 不从昨日日记拷贝任何条目到今日
- 不改写用户原话（即便措辞不规整）
