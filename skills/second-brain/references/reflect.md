# +reflect — 按需深度反思报告

## 语义

用户调用 `+reflect` 表示想深挖最近一段时间的素材，拿到一份**带反问**的报告。这是按需反思，区别于 `+checkup` 的月度体检。

## 参数

- `--window <span>`（可选，默认 `2w`）：覆盖的时间窗口。支持 `1w` / `2w` / `1m` / `3m` / `90d` 等表达。

## 执行流程

### Step 1 — 解析参数 + 计算时间窗

- 解析 `--window`，默认 `2w`
- 计算起止日期：今天往前推 `<span>`
- 列出窗口覆盖的月份（如 `2w` 从 5-11 往前到 4-27，覆盖 `raw/2026-05.md` 和 `raw/2026-04.md`）

### Step 2 — 加载素材（双源）

- 用 Read 工具读取 `~/second-brain/profile/` 全部 4 个文件（任一文件不存在时跳过，不报错）
- **raw 源**：用 Read 工具读取窗口覆盖的所有 `raw/YYYY-MM.md`，在 raw 内容里**只筛选窗口起止日期范围内的节**（按 `## YYYY-MM-DD` 时间戳判定）；**跳过标题含 `[元对话]` 的节**——那是协议谈判记录，不属于反思素材
- **daily 源**：用 Read 工具读取窗口覆盖的所有 `~/second-brain/daily/YYYY-MM-DD.md`，文件名本身即时间戳，整文件即素材
- **合并**：按时间戳排序成统一序列。复盘段（日记里的"📝 今日复盘"）视为当天 23:59；raw 段按各自的 `## YYYY-MM-DD HH:MM`。不去重——复盘是事实+改进、raw 是反思+情绪，两维度信息共存。

如果**两个源在窗口内都为空** → 立刻告诉用户："这个窗口内没有素材，要不要扩大窗口（比如 `--window 1m`）或者先 +chat / 开工攒点东西？"，**不要硬挤报告**。

只有一源有内容时正常生成报告（不报错），但在报告"素材"标注里说明（如"N 段对话 + M 篇日记"或"仅 N 段对话，本窗口内无日记"）。

### Step 3 — 生成报告

报告文件路径：`~/second-brain/reports/YYYY-MM-DD-reflect.md`（基于今天）。

如果同一天已经跑过 `+reflect`，追加 `-2`、`-3` 后缀以免覆盖（`YYYY-MM-DD-reflect-2.md`）。

**报告必须包含的 4 节**：

```markdown
# 深度反思报告 · YYYY-MM-DD

> 窗口：YYYY-MM-DD 至 YYYY-MM-DD（最近 <span>）
> 素材：N 段对话 + M 篇日记

## 1. 我看到的

> 中性的观察陈述——把这个窗口里反复出现的主题、情绪、人事物列出来。
> 不下判断，只描述。

-

## 2. 反复出现的主题

> 把"我看到的"聚类——哪些事情其实是同一件事的不同切面？
> 给每个主题一个名字 + 一段说明 + 引用具体的对话日期作证据。

### 主题 A：[名字]

[说明]

证据：YYYY-MM-DD（[一句话回忆]）、YYYY-MM-DD（...）

### 主题 B：[名字]

...

## 3. 我看到的矛盾点

> 用户在不同时候说过的相互矛盾或相互拉扯的话。
> 不是要批评，是要让用户自己看到。

- 你 YYYY-MM-DD 说 "X"，但 YYYY-MM-DD 又说 "Y"——这两个怎么共存？
-

## 4. 几个尖锐问题（留给你想，不要现在回答）

> 3-5 个反问。要具体、要锋利、要基于上面的观察。
> 不要泛泛的"你怎么看？"

1.
2.
3.
```

### Step 4 — 不动 profile

`+reflect` **只读不写 profile**。即使观察到看似新的 pattern，也不在这里提议更新——那是 `+checkup` 的职责。

### Step 5 — 完成后

向用户简短报告：
- 报告路径
- 主要发现一两句话（不剧透具体反问，让用户自己去看）

## 边界情况

- **窗口内 raw + daily 都为空**：见 Step 2，不要硬写。
- **窗口内素材极少**（< 3 段对话 + 日记总计）：报告可以短，但仍要包含 4 节结构——每节内容相应简短。
- **跨月**：把跨月的对话和日记合并按时间排序处理，不按月分别处理。
- **只有 daily 没有 raw**：合法。报告 4 节正常生成，但"主题"和"反问"会更偏向事实层——这是可接受的。
