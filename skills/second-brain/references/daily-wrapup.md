# 收工 — daily 段终结 + 教练衔接

## 语义

用户说"收工" / "下班"= 这一天的工作流要收尾。Agent 仍是秘书模式，做三段式复盘草稿；写入文件后**询问**是否切教练（不自动衔接）。

## 启动流程

1. **加载 profile**（用 Read 工具，顺序如下；任一文件不存在时跳过，不报错）：
   - `~/second-brain/profile/user.md`
   - `~/second-brain/profile/soul.md`
   - `~/second-brain/profile/patterns.md`
   - `~/second-brain/profile/memory.md`
2. 读今日 daily 文件：`~/second-brain/daily/YYYY-MM-DD.md`。不存在 → 告诉用户"今天还没开工，要不要先列一下今天做过什么、然后做复盘？"，用户愿意就走简化版（直接补今日 [x] 条目再复盘）。

## 拉清单

把当前 daily 文件中三类内容**原样贴出**（[ ] 和 [x] 一起，按三类分区）。某类整天无内容就空着，不脑补"无"。

示例：

```markdown
### 💻 工作
- [ ] (TODO) 调研 B 方案可行性
- [x] (10:23) EKS MCP 接口替换
- [x] (16:50) Family-Assistant 文档收尾

### 📚 学习
- [x] (14:05) 读完《白夜行》第 3 章

### 🏠 生活
- [x] (19:40) 跑 5km
```

贴完询问用户确认 / 补充。

## 推荐复盘事项

按重要性推荐 1-3 件值得复盘的事。判断依据：

- 计划内未完成（为什么没做？）
- 计划外完成（为什么临时加？）
- 描述里有情绪 / 想法子 bullet 的（白天没展开的）

用户可以指定不在推荐列表里的事。

## 三段式复盘草稿

对每一件复盘事，按模板写：

```markdown
### 事项 N：{事项名称}

**事实：**
- 计划了什么
- 实际完成了什么
- 关键时间节点

**原因 / 分析：**
- 为什么会有差异
- 原因是什么（客观 / 主观）
- 有什么意外发现

**改进措施：**
- 下次怎么做
- 需要沉淀到什么文件 / 流程
```

要点：

- **所有事项一次性给出**，不分批
- 以 markdown 形式贴在对话里
- 结尾加："以上复盘草稿请用户过目，有需要修改 / 补充 / 删减的地方直接说，确认无误后我再写入今日 daily 文件。"

## 用户确认

- 用户说"OK / 可以 / 写入吧" → 进入下一步
- 有修改 → 改完再发一版 → 继续等确认（可多轮）
- **草稿未确认前禁止写文件**（daily 红线 6）

## 写入 daily 文件

用 Read 工具读当前文件，用 Write 工具整体重写：在原内容末尾追加：

```markdown

## 📝 今日复盘

{用户确认后的三段式内容}
```

## 询问反思（衔接 +chat）

写入复盘段后，**先看当日 daily 文件有没有 `## 💬 待讨论` 区且非空**：

- **有且非空** → 问法变成（把待讨论条目逐条列出）：

  > "今天的复盘已写入。你白天提了几条想调整的协议——
  > - (15:42) {原话}
  > - (16:10) {原话}
  >
  > 要不要现在切教练一起过？"

- **没有 / 区为空** → 问法维持原版：

  > "今天的复盘已写入。要不要开始反思？"

用户回应：

- 用户说"开始反思" / "+chat" / "好的" / "过一下" → 进入 [`chat.md`](chat.md) 流程
- 用户说"不用了" / "明天再说" / "结束" → 进入下方"独立 memory 判断"

**用户拒绝时**：待讨论区**留在 daily 文件里不动**。本 skill 不做跨日迁移、不主动扫描历史 daily——如果用户日后想起来要处理，可手动打开历史文件、或主动 `+chat` 时手动指引。

## 独立 memory 判断（仅用户拒绝反思时走）

**判定问题**：今天的 daily 日记内容（任务 + 复盘）是否改变了"用户最近在忙 / 烦 / 状态"的整体画面？

- 改变了 → 用 Write 工具**重写整个 `profile/memory.md`**（不是追加；memory 永远只反映"现在"），更新 `## 上次更新` 为今天
- 没改变 → 不动

重写时保留模板的章节结构（在忙 / 在烦 / 情绪基调 / 上次更新）。

**重要**：如果用户走了"开始反思"分支，memory 判断由 `chat.md` 的四步提炼 Step 2 处理，**这里不再走**，避免重复写。

## 完成报告

简短报告给用户：
- daily 文件已追加复盘段
- memory 是否更新（一句话）
- 是否进入了反思（如进入则交给 chat.md 接管）

## 边界情况

- **a. 三类任务整天都无、复盘无可推荐**：仍可问用户"今天虽然没列任务，但有什么想聊的？"，用户说有就引到 `+chat`；说没有就只走独立 memory 判断
- **b. 用户说"不用复盘了，直接结束"**：跳过复盘草稿步骤，但仍走独立 memory 判断
- **c. 用户确认复盘草稿后又改主意要修改**：可再次进入修改循环
- **d. 写入时发现今日文件意外缺失**（不应发生，但兜底）：用 Write 工具新建文件，把清单 + 复盘段一起写入

## 不做什么（红线提醒）

- 不在复盘草稿未确认前就追加到文件
- 不擅自把 `+chat` 触发当默认（必须用户显式说"开始反思"或等价词）
- 不在 memory 判断里加历史信息（memory.md 只反映"现在"）
