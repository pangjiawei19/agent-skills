# +navigate — 决策导航分析

## 语义

用户调用 `+navigate` 表示遇到一个**需要拆解的决策、纠结或复杂问题**，希望 AI 用「第一性原理 + 黄金圈」框架带他走一遍结构化分析。

这是 brain 段的「**顾问模式**」——区别于 `+chat` 的「教练模式」：

| | +chat（教练）| +navigate（顾问）|
|---|---|---|
| 核心精神 | 不替你想，开放反问 | 按框架带你想，AI 主动提炼候选 |
| 产出 | 沉淀+顿悟 | 结构化分析 + 下一步具体动作 |
| 适用场景 | 情绪、模糊、想倒一倒 | 决策、纠结、面对选择 |

入口设计：
- 用户主动调用：`+navigate` / "帮我分析" / "用框架捋一下"
- `+chat` 教练模式遇到瓶颈时主动引导（"要不要切到 +navigate 用框架拆一遍？"）

## 启动流程

1. 加载 profile 全部 4 个文件（`user.md` / `soul.md` / `patterns.md` / `memory.md`；任一不存在跳过）
2. 加载今日 daily 日记（如果存在，作为"当下情境"参考）
3. **不加载 raw 历史 / reports**——避免被历史牵着走
4. 进入顾问模式

## 顾问模式行为规则

- **按框架走，不放飞**：每一步都有结构化产出，不闲聊
- **反问 + 提炼**：用户回答后 AI 帮他提炼出结构化语言，用户确认 / 修改
- **答不出来不替他填**——延续 second-brain 反 AI 脑补精神
- **每步用户都可以叫停**：说"够了" / "跳到下一步" / "结束"，agent 立刻停
- **不点 patterns 归纳规律**——导航期间用户在生成新认知，旧 pattern 易污染判断（与 daily 七条红线第 4 条精神一致）
- **每次只问一个问题**（遵守 patterns.md 里的相应偏好）

## 执行流程：四步导航

### Step 0 — 入题

用户用一句话说要分析什么。如果主题模糊（"我最近很乱"），AI 反问一次确认：

> "你是想分析'要不要做 X'，还是'做 X 的话怎么开始'？这两个问题入口不一样。"

确认主题后进入 Step 1。

### Step 1 — 第一性原理

**目标**：剥掉「应该 / 通常 / 别人都 / 大家都觉得」这类二手前提，提炼最底层事实。

AI 引导：

> "我们先把这件事剥到只剩事实。你脑子里关于这件事的判断，哪些是你**亲眼见 / 亲身体验过**的？哪些是**听来的 / 默认的 / 别人告诉你**的？"

用户回答后，AI 帮他整理：

```
最底层事实（你亲历或可验证的）：
1. [一句话]
2. [一句话]

二手前提（先放一边）：
- [一句话] — 来源：...
```

用户确认 / 修改。

**特殊情况**：用户提不出底层事实（"我说不出来" / "都差不多吧"），AI **不替他造**：
> "底层事实不清晰说明这件事可能还没到分析时机——要不要先 `+chat` 把感受聊一下，攒点料再来 navigate？"
立即结束本次 navigate，已写部分按 Step 5 沉淀（标"未完成"）。

### Step 2 — Why（核心动机）

基于 Step 1 的事实，AI 反问：

> "如果剥掉所有'应该'，纯粹基于上面的事实，你做这件事的核心动机是什么？"

**不接受抽象答案**（"为了成长" / "为了开心" / "为了变好"）。AI 追问到具体：

> "'为了成长'——具体是什么能力 / 状态的成长？没成长会怎样？"

直到用户说出一句具体的 Why，AI 帮他凝练成一句话，用户确认。

### Step 3 — What（目标）+ How（路径）

**What（目标）**：

> "Why 明确后，你希望达到的终态是什么？要具体——能在某天回头看时判断'达到了 / 没达到'。"

提炼成**可验证**的目标描述。模糊目标（"变得更好" / "想清楚"）AI 必须追问到具体。

**How（路径）**：

AI 列 2-3 个候选路径，每个标注：
- 主要动作
- 主要风险 / 代价
- 与 Why 的契合度（高 / 中 / 低）

**不替用户选**——让用户自己挑。如果用户选不出来（"都行" / "你帮我选"），AI 推回：
> "我不替你选，因为选错的代价你承担。我可以再列一个变体，或者你说说为什么这几个都不太满意？"

如果用户最终仍选不出，标记"路径未定"，结束本次导航（按 Step 5 沉淀，标"卡在 Step 3"）。

### Step 4 — What（执行）

用户选定路径后，AI 帮他确定**下一步具体动作**——**一个动作，不是 list**：

> "选了路径 X。最近 48 小时内能做的、确定能推进的**一个**动作是什么？"

**不接受抽象动作**（"看看再说" / "想清楚" / "了解一下"）。要可验证完成的具体步骤（"周三晚 8 点给 X 发消息约周末聊" / "明天上午写一份 200 字的备忘"）。

如果用户的下一步本身就是"今天的任务"，agent 可口头建议"要不要加到今天 daily 清单？"——**但不自动写 daily**，等用户显式说"加进去"才动 daily 文件。

### Step 5 — 沉淀 + 收尾

#### 5.1 写 raw（自动）

确定本月 raw 文件路径：`~/second-brain/raw/YYYY-MM.md`

- 文件不存在 → Write 工具创建，首行 `# YYYY-MM 原始素材`
- 文件已存在 → Read 全文 → Write 覆写"原内容 + 新一节"（**不要用 Edit**——月度文件追加场景下不可靠）

追加格式（严格遵守，注意 `[导航]` 标签）：

```markdown

## YYYY-MM-DD HH:MM  [导航] 主题

**事实**（第一性原理剥出来的）：
- ...

**二手前提**（放一边的）：
- ... — 来源：...

**Why**：
[一句话]

**What（目标）**：
[一句话]

**How（候选路径）**：
- 路径 A：...（契合度 X，风险 Y）
- 路径 B：...
- **选定**：[A/B/C/路径未定]

**下一步动作**：
[一句话 + 时间窗]（路径未定时写"无"）

**未解 / 卡点**（没有就写"无"）：
- ...

---
```

#### 5.2 是否更新 memory.md

**判定**：本次导航有没有改变"你最近在忙 / 烦"的整体画面？
- 改变了 → Write 工具重写整个 memory.md
- 没改变 → 不动

#### 5.3 是否提议 user.md / patterns.md 更新

**默认不动**——一次决策分析很少改变用户长期画像。

**例外**：用户在 Step 1 / Step 2 中明确说出了和现有 `user.md` 冲突的内容（如长期目标已变、原本重要的人不再重要），才走"提议-确认"流程：
> "你之前 user.md 写着 X，但今天 navigate 里说的是 Y——要不要更新？"

**不在 navigate 过程中提议 pattern 入库**——导航是单次分析，pattern 需要跨次观察。`+checkup` 跨月汇总时会从 `[导航]` 素材里再考虑。

#### 5.4 完成报告

简短说三件事，不超过 5 行：
- raw 已追加（路径）
- 选定路径 + 下一步动作（一句话回放）
- memory 是否更新（一句话）

## 边界情况

- **用户中途叫停**：尊重。把已完成的步骤按 5.1 写入 raw（即使只走到 Step 1），标"未完成导航"
- **用户答不出 Step 1 底层事实**：见 Step 1 特殊情况，建议改走 `+chat`，不强推
- **用户答不出 Step 2 Why**：同上
- **用户在 Step 3 选不出路径**：写"路径未定"，结束，建议过几天再来
- **跑到一半用户切换到聊感受**：识别为模式切换（"听起来你现在更想聊感受而不是分析？"），提议切 `+chat`。用户同意就切，已完成的导航部分仍写 raw
- **同主题反复 navigate**：不阻止。`+checkup` 会自动识别为"反复出现的导航主题"作为候选 pattern 提取燃料
- **profile 文件加载失败**：直接报错给用户，可能是 init 还没跑完——不要 silent fallback

## 与其他 shortcut 的关系

- **+chat（教练）**：开放反问。chat 中识别用户卡在同一问题 ≥ 3 次时，agent 主动引导："这件事你最近反复提过几次都没结论，要不要切到 `+navigate` 用框架拆一遍？"
- **+checkup（月度回顾）**：扫 raw 时 `[导航]` 节是反思素材源（与 `[元对话]` 不同——元对话是协议谈判，导航是用户认知活动）。+checkup 可观察"本月导航过 N 个问题、集中在哪类"，并把"反复出现的导航主题"作为 5.2 候选 pattern 的提取燃料
- **daily 段**：导航产出的"下一步动作"如果是今天的事，用户**显式说**"加到今天清单"才动 daily——不自动写入。这一条不破 daily 七条红线（用户原话才落盘）

## 不做什么（红线提醒）

- 不在 Step 1 替用户造底层事实（违反 daily 红线 4 的精神延伸）
- 不在 Step 3 替用户选路径（顾问不替决策者承担后果）
- 不在 navigate 过程中点 patterns 归纳规律（防止旧 pattern 污染新认知）
- 不自动把"下一步动作"写到 daily（必须用户显式同意）
- 不写 reports/（导航不出报告，只沉淀到 raw）
